CORRECTNESS (25 pts) — register of an already-registered user is wrongly accepted

ROOT CAUSE
The reference rejects 'register' for a username that already exists: it returns
"invalid" and exit 255 (starter-package/build/stor.c:333-335). This target instead
RE-REGISTERS the existing user (overwriting their salt/key verifier) and returns
exit 0.

SELF-CONTAINED TEST (no enc.db seeding needed)
  cmd[0]: register atk with key pw1  -> both oracle and target: exit 0
  cmd[1]: register atk with key pw2  -> oracle: "invalid"/255 ; target: exit 0
cmd[1] outputs differ -> valid correctness break.
