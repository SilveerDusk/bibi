Target team: 108
Bug title: write succeeds on a tampered enc.db instead of rejecting corruption
Bug type: integrity
Source testcase: security_focus_tests.sh J10_tampered_db_rejects_write

A single-byte tamper of enc.db should cause subsequent authenticated write to
fail. stor108 accepts the write with exit 0.

Minimal reproducer:
  stor -u alice -k k register
  stor -u alice -f notes create
  stor -u alice -k k -f notes write before_tamper
  python3 -c "d=bytearray(open('enc.db','rb').read()); d[-2]^=0x7; open('enc.db','wb').write(d)"
  stor -u alice -k k -f notes write after_tamper

Observed: write exit 0
Expected: stdout=invalid, exit 255
